In general, many times we are faced with a situation where we need to run a time recorder that constantly shows us the current time or keeps executing a certain amount of code peacefully for a given time interval, in this case we should use Ticker, to use this we need to use the time package of the Golang, we have a method called NewTicker(). It allows us to stop and start the time code, we need to create a code channel by passing chan and bool as parameters that will be used to check if it is open, if the channel is open means that the timer will continue.

Go Timer

Go provides a very simple syntax to implement a timer with the following structure definition.

1
2
3
4

type Ticker struct {
	C <-chan Time // The channel thrown out to the upper system to use to implement the timing
	r runtimeTimer // timer to be used by the system management, the system uses this field to determine if the timer is up, and if it is, call the corresponding function to push the current time to C.
}

Ticker exposes only one channel to the public and writes the system time to that channel when the specified time comes.

The Ticker is also very simple to use, with the following codeï¼š

import time

TimeTicker := time.NewTicker(1 * time.Second)

TimeTicker.Stop()
  • time : If you want to use the timer in time, then you need to import it into the code
  • NewTicker() function: this is easy to understand, it creates a new timer and then the timer takes a time expression as an argument, for example 1*time.Second, instead of passing 1 directly, you can also pass other time methods, for example 2*time.Millisecond
  • Finally, we can use a time expression to call any activity on a variable created by the NewTicker function. For example, in the above syntax, we used TimeTicker.Stop() to stop the time timer. We can use it under specific conditions, like defining a timer that counts down to 10 seconds to check if the time meets 10 seconds, and once 10 seconds occurs, we can call TimeTicker.stop().

How does the Ticker timer work

The timer created by NewTicker and the timer created by NewTimer hold the same time channel with a cached channel, and the function executed after each trigger is also sendTime, which ensures that the Ticker will not block when triggering time events, regardless of whether there is a receiver or not.

func NewTicker(d Duration) *Ticker {
    if d <= 0 {
        panic(errors.New("non-positive interval for NewTicker"))
    }
    // Give the channel a 1-element time buffer.
    // If the client falls behind while reading, we drop ticks
    // on the floor until the client catches up.
    c := make(chan Time, 1)
    t := &Ticker{
        C: c,
        r: runtimeTimer{
            when:   when(d),
            period: int64(d),
            f:      sendTime,
            arg:    c,
        },
    }
    startTimer(&t.r)
    return t
}

NewTicker() just constructs a Ticker, and then hands over Ticker.r to the system concurrently for maintenance via startTimer(). Where period is the period of the event trigger. The sendTime() method is the action when the timer is triggered.

Stop Ticker

Stops Ticker and simply removes Ticker from the system concurrent. The main implementation of the function is as follows.

1
2
3

func (t *Ticker) Stop() {
    stopTimer(&t.r)
}

stopTicker() notifies the system concurrent that the Ticker is removed and no longer monitored. The system concurrent process only removes the Ticker and does not close the pipeline to avoid reading errors by the user concurrent process

package main

import (
	"fmt"
	"time"
)

func main() {
	TimeTicker := time.NewTicker(3 * time.Second)
	tickerChannel := make(chan bool)
	go func() {
		for {
			select {
			case timeticker := <-TimeTicker.C:
				fmt.Println("The time for current is : ", timeticker)
			case <-tickerChannel:
				return
			}
		}
	}()
	time.Sleep(6 * time.Second)
	TimeTicker.Stop()
	tickerChannel <- true
	fmt.Println("Time for running ticker is completed")
}

package main

import (
	"fmt"
	"time"
)

func main() {
	tm := time.Millisecond
	tickerTicker := time.NewTicker(400 * tm)
	tickerChaneel := make(chan bool)
	go func() {
		for {
			select {
			case <-tickerChaneel:
				return
			case tmtr := <-tickerTicker.C:
				fmt.Println("Ticker time at current is", tmtr)
			}
		}
	}()
	time.Sleep(1400 * time.Millisecond)
	tickerTicker.Stop()
	tickerChaneel <- true
	fmt.Println("Ticker has stopped now")
}

Summary

  • Using time.NewTicker() to create a timer.
  • Use Stop() to stop a timer.
  • releasing a timer when it is finished being used, otherwise a resource leak will occur.
  • NewTicker() to create a new Ticker to be handed over to the system concurrently for monitoring.
  • Stop() notifies the system concurrent process to delete the specified Ticker;
